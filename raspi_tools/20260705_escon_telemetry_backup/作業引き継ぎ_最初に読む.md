## まず結論

現時点では、電流値用のアナログ入力はESP32で読めています。

一方で、回転数用に使っている ESCON の Analog Output 2 は、実機確認では負電圧を出している可能性があります。ESP32のADCは負電圧を読めないため、モータが回っていてもESP32側の `rpm` が `0` になることがあります。

そのため、今の `rpm=0` を「モータ停止」と判断しないでください。

## 現在見えている問題

### 1. 回転数モニタがESP32で正しく読めない可能性

ESCON Studioで確認したとき、実回転数が負方向になっており、AO2が負電圧になっているように見えました。

ESP32 ADCは0V未満を読めないため、AO2を以下のように0から3.3Vに収まるよう設定する必要があります。

例:

- `-9690 rpm -> 0.0 V`
- `0 rpm -> 1.65 V`
- `+9690 rpm -> 3.3 V`

この設定にできた場合、ESP32側の計算式は以下の考え方に変更します。

```cpp
rpm = (speed_v - 1.65f) * (9690.0f / 1.65f);
```

### 2. 逆転指令時に電流・回転数が出ていない

ラズパイから逆転方向の指令を送ったログでは、`enabled=1` になっているのに、`current_v=0`, `speed_v=0` のままでした。

次に確認すること:

- ESP32 `GPIO23` が ESCON の `DI2` に届いているか。
- ESCON Studioのモニタで、逆転指令時に `DI2` が変化しているか。
- DI2 の設定が本当に `Rotation direction` になっているか。
- DI2 の論理が反転していないか。

### 3. `encoder_rpm` が常に0

ESP32側のエンコーダ読み取りは、現状まだ動作確認できていません。

次に確認すること:

- エンコーダA/B相の配線
- ESP32側の入力ピン
- プルアップ/プルダウン
- 信号が単相か、2相か、差動か、オープンコレクタか
